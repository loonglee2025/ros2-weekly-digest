# ROS2 Weekly Digest — 2026-06-15 至 2026-06-21

> **总判断**：本周 ROS2 社区延续 Lyrical GA 后的消化期，核心框架代码活跃度极低，但外围生态与工具创新异常密集。最值得提高跟踪权重的三个方向是：**Jazzy Patch Release 7 与 Autoware 大规模入驻**（97 个新包）、**纯 Rust RMW（ZeroDDS）进入 rc.3**、以及 **Gazebo WebSocketServer 安全漏洞的完整修复与披露**。社区正从“功能可用”加速向“可观测、可调试、安全加固”转型，AI 辅助调试工具（Scry）和 DDS 图形化调试工具（CycloneDDS Insight）同步涌现，说明运维体验已成为生态竞争的新焦点。

## 本周关键信号

1. **Jazzy Jalisco Patch Release 7 发布，Autoware 核心栈大规模入驻**（6 月 18 日）
   - 本次 sync 包含 **97 个新包** 和 **608 个更新包**，是近几周规模最大的 Jazzy 发布。最引人注目的是 **Autoware 核心模块首次大规模进入 Jazzy 官方仓库**——从 `autoware_core` 到 `autoware_behavior_velocity_planner`、`autoware_ndt_scan_matcher`、`autoware_ekf_localizer` 等约 60+ 个包一次性入驻。工程意义：这标志着 ROS2 Jazzy 正式成为 Autoware 的主推基线，自动驾驶开源生态与 ROS2 发行版的绑定深度显著加强。对于使用 Jazzy 的自动驾驶团队，这意味着可以直接通过 `apt` 安装 Autoware 核心模块，而不必维护复杂的源码构建链。需继续跟踪 Autoware 在 Jazzy 上的 CI 稳定性、与原生 ROS2 工具链的兼容性，以及是否有性能回归。

2. **ZeroDDS 发布 rc.3：纯 Rust 编写的替代 RMW，基于 349 条真实痛点设计**（6 月 19 日）
   - ZeroDDS 是一个从头用 Rust 编写的 DDS 栈（Apache-2.0，无 per-seat 授权），提供原生 `rmw_zerodds_cpp` 实现，可与现有 CycloneDDS/Fast DDS 节点双向互操作。其设计直接针对社区收集的 349 条真实 ROS↔DDS 痛点：静默 QoS/类型不匹配（已修复 keyed/no-key entity-id 场景）、WiFi 发现丢失（初始通告 burst 机制）、大规模集群/WAN（unicast-only `multi_robot()` profile）、DDS-Security/SROS2 支持（消费 SROS2 enclave 文件，配置错误时硬失败而非静默降级）。rc.3 新增 Windows 零拷贝共享内存支持。工程意义：这是 ROS2 中间件层首次出现纯 Rust 的 Tier-2 候选，其内存安全特性对长期运行系统具有结构性价值。但作者明确声明尚未安全认证、OMG Vendor ID 仍在申请中，性能与成熟 DDS 栈持平而非全面领先。下一步应关注其持续多发行版 smoke CI（rc.4 计划）的覆盖结果，以及 ROS2 安全变量（`ROS_SECURITY_KEYSTORE`）的原生集成进度。

3. **Gazebo WebSocketServer 安全漏洞完整披露与修复**（6 月 18 日）
   - `WebsocketServer` 插件的 `asset` 操作存在**任意文件读取漏洞**（gz-sim#3589），攻击者可通过未经验证的绝对路径访问主机文件系统。默认无 `<authorization_key>` 配置时，任何能访问 WebSocket 端口（9002）的网络客户端均可利用。修复方案：将资源读取限制在已批准的资源列表（Gazebo resource 目录和 Fuel cache）内。已 backport 到所有支持版本：ign-launch 5.3.1（Fortress）、gz-launch 7.1.2（Harmonic）、gz-launch 8.0.3（Ionic）、gz-launch 9.0.1 / gz-sim 10.4.0（Jetty）。工程意义：这是 Gazebo 生态近期最严重的安全事件之一，且影响范围跨越 Ignition→Gazebo 的架构历史（旧版 `ign-launch` 也受影响）。建议所有使用 Gazebo WebSocket 流式传输的团队立即确认版本并升级。Gazebo 官方同时发布了 `security@openrobotics.org` 私有报告通道和 `SECURITY.md` 政策，安全响应流程正在规范化。

4. **CycloneDDS Insight 发布：图形化 DDS 调试与拓扑可视化工具**（6 月 20 日）
   - Eclipse Cyclone DDS 官方发布 Insight，提供 DDS 参与者、主题、发布者/订阅者的实时图形化检视，支持多主机 ROS2 部署的通信架构可视化、发现问题诊断和流量统计监控。完全开源免费。工程意义：这是 Cyclone DDS 生态在可观测性上的重要补充，与上周的 ros2probe（eBPF 被动捕获）形成互补——Insight 面向交互式图形调试，ros2probe 面向无侵入高性能记录。对于使用 CycloneDDS 的团队，Insight 可显著降低多节点分布式系统的调试门槛。需关注其是否支持 ROS2 特定语义层（如 node 名解析、QoS 可视化）。

5. **Lyrical Sync 2026-06-22 准备中，174 个包待同步，1 个回归**（6 月 18 日）
   - sloretz 发布 Lyrical 同步准备帖，计划 6 月 22 日将 174 个包同步到主 apt 仓库，当前存在 1 个回归待处理。工程意义：Lyrical GA 后的首次大规模 sync，规模与节奏正常。需关注 sync 后是否有新的 regression 报告，以及 Autoware 包是否会在 Lyrical 的下一轮 sync 中出现。

6. **Scry 发布：AI 优先的手机端 ROS2 调试工具**（6 月 21 日）
   - Scry 是一个 AI-first 的 ROS2 手机调试工具（Android），通过 `scry-connect` Python 服务器与机器人通信，支持自然语言查询（如“为什么机器人不动了？”），自动检视 topics、nodes、services、parameters、lifecycle、TF、logs 并给出根因分析。支持 OpenRouter（300+ 模型）和本地 Ollama 离线运行，所有写操作需显式批准。覆盖 Humble 到 Rolling 全部发行版。工程意义：这是“AI + ROS2”在运维场景的具体落地，其无云架构（自有网络、无遥测）对工业现场部署是重要加分项。但早期阶段，真实机器人上的覆盖率和可靠性仍需社区验证。与 AgenticROS、rclnodejs 2.0.0 共同构成 AI+ROS2 的三条路线：运维调试（Scry）、Web 桥接（rclnodejs）、AI Agent 控制（AgenticROS）。

7. **MetriPlane v0.2.0：物理工作单元的可观测性与回归测试工具**（6 月 21 日）
   - 开源物理可观测工具，将 replayed workcell state 转化为物理事件、证据包和生成的回归测试。工程意义：虽然项目尚处早期（MSc  thesis 阶段），但其“物理事件 → 证据包 → 回归测试”的闭环思路对制造场景下的机器人系统验证有参考价值。与 ros2probe 类似，都属于 ROS2 生态从“功能开发”向“可观测、可验证”转型的信号。

## 工程化观察

- **回归风险**：Jazzy Patch Release 7 规模巨大（97 新包 + 608 更新），Autoware 核心栈首次大规模入驻，引入回归风险的可能性上升。建议 Jazzy 用户在升级后重点验证：Autoware 模块与原生 ROS2 工具链的兼容性、agnocast（零拷贝中间件）的稳定性、以及新增包对系统依赖的冲击。Lyrical 的 6 月 22 日 sync 存在 1 个已知回归，需关注 sync 后是否扩大。
- **安全态势**：Gazebo WebSocketServer 漏洞（CVSS 尚待评估，但预认证任意文件读取属于高危）的完整修复和 backport 展示了 Open Robotics 安全响应能力的提升。新设立的 `security@openrobotics.org` 和 `SECURITY.md` 是长期利好。建议所有使用 Gazebo WebSocket 流式传输的团队立即升级，并检查端口 9002 的防火墙规则。
- **可观测性工具爆发**：本周同时出现 CycloneDDS Insight（图形化 DDS 调试）、Scry（AI 手机端调试）、MetriPlane（物理工作单元回归测试），加上上周的 ros2probe（eBPF 被动捕获），ROS2 生态的可观测性工具正在经历一次小型爆发。这反映社区需求从“能不能跑”转向“出了问题怎么快速定位”。
- **中间件多元化**：ZeroDDS rc.3 是 ROS2 生态中第一个纯 Rust 的 RMW 实现，与 Zenoh、CycloneDDS、Fast DDS 形成四足鼎立雏形。其设计方法论（基于 349 条真实痛点）值得肯定，但生产就绪仍需时间。建议团队保持关注，暂不用于生产关键路径。
- **Windows 开发体验进一步收缩**：Jazzy Patch Release 7 完全跳过 Windows 二进制包，这是官方首次在 LTS 发行版中执行 Platform EOL 政策到“不发布”的程度。Windows 原生 ROS2 开发的可行性正在快速下降，建议相关团队评估向 WSL2 或 Linux 原生迁移的时间表。
- **Autoware 与 ROS2 发行版绑定加深**：约 60+ 个 Autoware 包进入 Jazzy 官方仓库，意味着 Autoware 的维护者将直接参与 Jazzy 的 sync 和回归测试流程。这对 Jazzy 的长期稳定性是利好，但也意味着 Jazzy 的 sync 节奏可能受到 Autoware 发布周期的影响。

## 前沿议题

1. **RMW 层进入“多极化”时代**
   - Fast DDS（默认）、CycloneDDS（Tier 1）、Zenoh（实验性 Tier 1）、ZeroDDS（Rust 新秀）四种实现同时活跃，各自面向不同场景。ZeroDDS 的内存安全和无 per-seat 授权对商业部署有吸引力，但生态成熟度和安全认证仍是短板。下一步应关注 ROS2 官方是否会在 REP 中明确新 RMW 的 Tier 评定标准。

2. **AI + ROS2 从“概念验证”进入“工具化”**
   - Scry（手机端 AI 调试）、AgenticROS（AI Agent 控制）、rclnodejs 2.0（Web 桥接）三条路线并行，说明 AI 与 ROS2 的集成不再停留在 demo 阶段，而是开始解决真实工程痛点（现场调试、自然语言交互、Web 暴露）。需关注这些工具在真实工业环境下的延迟、可靠性和安全审计结果。

3. **可观测性成为生态竞争新焦点**
   - ros2probe（eBPF 无侵入）、CycloneDDS Insight（图形化 DDS）、Scry（AI 手机端）同时出现，说明 ROS2 社区正集体跨越“功能可用”门槛，进入“可观测、可运维、可规模部署”阶段。这对大规模 Fleet 管理和工业场景是必经阶段。

4. **Autoware 生态与 ROS2 发行版的协同效应**
   - Autoware 大规模入驻 Jazzy 意味着 ROS2 发行版将成为自动驾驶开源软件的基础设施层。这可能推动 ROS2 在实时性、确定性、安全认证方面的专项优化，同时也对发行版的 sync 质量和回归测试提出更高要求。
